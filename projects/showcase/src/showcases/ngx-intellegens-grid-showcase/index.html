<ngx-intellegens-grid />

The ngx-intellegens-grid provides a custom table solution based on Angular Material.

The table comes with a set of predefined optional features which include filtering, sorting, pagination, internal and external error handling and loading.


Working with ngx-intellegens-grid


1. Add ngx-intellegens-grid component

Begin by adding the <ngx-intellegens-grid> component to your template.


2. Provide data to the template

The simplest way to provide data to your table is by passing it a data array.

<ngx-intellegens-grid [dataSource]="myArray"></ngx-intellegens-grid>

Examples


Basic ngx-intellegens-grid use case where the table uses an array as a data source.

The table also supports more complex data sources such as Observable streams and Promises

The table also supports custom headers and footers


Headers and footers can be modified by providing template bindings inside the <ng-container>
<ng-container ngxIntellegensGridColumnDef="location" header="Location" footer="Location"><ng-container>


If there is no explicit header and/or footer provided the table will automatically fall back to it's default value, which is shown in the example with the header of the "firstName" column.

Features


Internal and external error handling and loading

<ngx-intellegens-grid> uses build-in methods that support internal and external error detection and loading overlay and provides a simple handling solution for both of them.


To test the external error and loader on the example bellow click the "Toggle Error" and "Toggle Loading" buttons.


You can also test the internal loading feature by choosing "Observable" as the provided data source and pressing the "Reload Data" button to reload the table data.

This will trigger the loading overlay to be displayed over the table while the data loads.
