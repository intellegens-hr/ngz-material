[loading] and [error] attributes

Other than the internal "loading" and "error" states, which are triggered by dataSource changes, there are also external [loading] and [error] attributes which can be use to force those states directly. 

  
    let rows, loading, error;
    try {{'{'}}
      // Set loading status
      loading = true;
      // Get rows
      rows = await loadDataFromAsync();
      // Reset loading status
      loading = false;
    {{'}'}} catch (err) {{'{'}}
      // In case of error, capture error  
      error = err;
    {{'}'}}
    const rows = [...];
  
  
    <ngz-grid [dataSource]="rows" [loading]="loading" [error]="error"></ngz-grid>
